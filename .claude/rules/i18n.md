# Internationalization (i18n) Rules — MANDATORY

## Stack
- **Library:** `next-intl` v4
- **Supported locales:** `de` (default), `en`
- **Message files:** `src/messages/de.json`, `src/messages/en.json`
- **Navigation helpers:** `src/i18n/navigation.ts` (`usePathname`, `useRouter`, `Link`, `redirect`)
- **Routing config:** `src/i18n/routing.ts`

## CRITICAL: No Hardcoded User-Facing Strings

**NEVER do this:**
```tsx
<p>Willkommen zurück</p>
<Button>Speichern</Button>
<span>Settings</span>
```

**ALWAYS do this:**
```tsx
const t = useTranslations("namespace");
<p>{t("welcomeBack")}</p>
<Button>{t("save")}</Button>
```

## Adding New Strings

1. Add the key to **both** `src/messages/de.json` and `src/messages/en.json`
2. Use a logical namespace (e.g. `"dashboard"`, `"nav"`, `"auth"`, `"settings"`)
3. German strings must use correct umlauts: **ä, ö, ü, ß** — NEVER ASCII substitutes like ae, oe, ue, sz
4. In Server Components: `const t = await getTranslations("namespace")`
5. In Client Components: `const t = useTranslations("namespace")`

## Navigation

Always use locale-aware navigation from `@/i18n/navigation`:
```tsx
// ✅ Correct
import { usePathname, useRouter, Link } from "@/i18n/navigation";

// ❌ Wrong — loses locale prefix
import { usePathname, useRouter } from "next/navigation";
import Link from "next/link";
```

`usePathname()` from `@/i18n/navigation` automatically strips the locale prefix.
So `/de/dashboard` → `/dashboard` for active-state comparisons.

## Page Routes

All pages live under `src/app/[locale]/`. Never create pages at the root `src/app/` level (except `globals.css` and other non-page files).

## Localized Pathnames (MANDATORY)

German-named routes **MUST** have English URL equivalents configured in `src/i18n/routing.ts` via the `pathnames` option. The filesystem path stays German, but the URL is translated per locale.

**Current mappings:**
| Filesystem path | DE URL | EN URL |
|---|---|---|
| `/datenschutz` | `/datenschutz` | `/privacy` |
| `/impressum` | `/impressum` | `/imprint` |
| `/agb` | `/agb` | `/terms` |
| `/account/datenschutz` | `/account/datenschutz` | `/account/privacy` |

**When adding new German-named routes:**
1. Add the pathname mapping in `src/i18n/routing.ts` → `pathnames`
2. Add both DE and EN path variants to `PUBLIC_ROUTES` in `src/middleware.ts` (if the route is public)
3. Use `Link` from `@/i18n/navigation` — **NEVER** hardcode `href={`/${locale}/path`}`
4. The `Link` component automatically translates paths based on the current locale

**NEVER do this:**
```tsx
<a href={`/${locale}/datenschutz`}>Datenschutz</a>
```

**ALWAYS do this:**
```tsx
import { Link } from "@/i18n/navigation";
<Link href="/datenschutz">{t("privacy")}</Link>
```

## Message File Structure

Organize by feature/namespace. Example:
```json
{
  "nav": { ... },
  "sidebar": { ... },
  "header": { ... },
  "userMenu": { ... },
  "dashboard": { ... },
  "auth": { ... },
  "athletes": { ... }
}
```

## German Umlaut Reference

| Wrong (ASCII) | Correct |
|---|---|
| ae | ä |
| oe | ö |
| ue | ü |
| sz / ss | ß |
| Ueberblick | Überblick |
| Ernaehrung | Ernährung |
| Koerpermasze | Körpermaße |
| zurueck | zurück |

## Metadata (Page Titles)

Use `generateMetadata` with `getTranslations` for SEO-correct metadata per locale:
```typescript
export async function generateMetadata({ params }: { params: Promise<{ locale: string }> }) {
  const { locale } = await params;
  const t = await getTranslations({ locale, namespace: "metadata" });
  return { title: t("title"), description: t("description") };
}
```
